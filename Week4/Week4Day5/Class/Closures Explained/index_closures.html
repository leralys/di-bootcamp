Definition of closure

A closure is a function having access to the parent scope even after a parent function has closed

Closures keep local variables alive from functions that should have destroyed them a long time ago.

We are doing a counter, when we click on the button, we increment the counter by 1

Here is the process to do this exercise

  1. We use a global variable, but the problems are:
    • global variables stay in the memory. The garbage collector cannot delete them
    • global variables are public, any functions or program can modify them

  2. To avoid using global variable, let's use local variables,but the problems are:
    • the local variable keeps being resetted to 0

  3. To avoid using local variable, we could try nested functions, but the problems are:
    • To reach the inner function, we need to call it inside the outer function. So the incrementation is not dynamic
    • The local variable keeps being resetted to 0 because the outer function is invoked everytime we click on the button

  4. We could use closures, the benefits are:
    • We have access to the inner function, after the outer function has been executed
    • We can invoke the inner function in the global scope
    But the problem is:
    • The local variable keeps being resetted to 0 because the outer function is invoked everytime we click on the button

  5. SOLUTION: we need to use closures and self invoking functions. The benefits are:
    • We call the outer function ONCE, so the counter is set to 0 ONCE.
    • Everytime we click on the button, we call ONLY the inner function
